## 📂 Структура проєкту та опис робіт

Проєкт структуровано за окремими директоріями (папками) для кожної лабораторної роботи. Кожен модуль містить задокументований вихідний код (`.java`), що реалізує бізнес-логіку відповідно до розрахованого варіанта.

### 🔢 Лабораторна робота №0: Основні типи та оператори
* **Варіант:** 17
* **Формули варіанта:** * $C_2 = 17 \pmod 2 = 1$ (Операція 01: `-`)
  * $C_3 = 17 \pmod 3 = 2$ (Константа $C = 2$)
  * $C_5 = 17 \pmod 5 = 2$ (Операція 02: `%`)
  * $C_7 = 17 \pmod 7 = 3$ (Тип індексів: `int`)
* **Опис завдання:** Обчислення значення функції $S = \sum \sum ((i \pmod j) / (i - C))$ із заданим типом індексів. Реалізовано обробку виключних ситуацій (ділення на нуль при $i = C$ та $j = 0$).

### 📊 Лабораторна робота №1: Масиви та матриці
* **Варіант:** 17
* **Формули варіанта:**
  * $C_5 = 17 \pmod 5 = 2$ (Дія з матрицями: $C = A + B$)
  * $C_7 = 17 \pmod 7 = 3$ (Тип елементів матриці: `int`)
  * $C_{11} = 17 \pmod{11} = 6$ (Дія з результуючою матрицею)
* **Опис завдання:** Поелементне додавання двох матриць типу `int`. В отриманій матриці $C$ обчислюється сума найбільших елементів у стовпцях з парними номерами та найменших елементів у стовпцях з непарними номерами.

### 📝 Лабораторна робота №2: Обробка текстових рядків
* **Варіант:** 17
* **Формули варіанта:**
  * $C_3 = 17 \pmod 3 = 2$ (Тип текстових змінних: `StringBuffer`)
  * $C_{17} = 17 \pmod{17} = 0$ (Дія з текстом)
* **Опис завдання:** Робота з текстом за допомогою класу `StringBuffer` без використання заборонених перетворень у `String`. Програма аналізує текст та знаходить найбільшу кількість речень, у яких зустрічаються однакові слова.

### 🚴 Лабораторна робота №3: Класи та об'єкти
* **Варіант:** 17
* **Формули варіанта:**
  * $C_{11} = 17 \pmod{11} = 6$ (Клас: Спортивний інвентар)
* **Опис завдання:** Створення користувацького класу `SportsEquipment` із щонайменше 5 полями. Реалізовано стандартні методи (`equals`, `toString`) та сортування масиву об'єктів за допомогою `Arrays.sort` за двома полями: спочатку за зростанням ціни, а при рівності — за спаданням ваги. Виконано пошук ідентичного об'єкта.

### 🧱 Лабораторна робота №4: Композиція класів
* **Варіант:** 17
* **Опис завдання:** Декомпозиція та рефакторинг лабораторної роботи №2. Створено окремі класи високого рівня абстракції: `Letter` (Літера), `Word` (Слово — масив літер), `Punctuation` (Розділовий знак), `Sentence` (Речення — масив слів та розділових знаків) та `Text` (Текст — масив речень). Програма замінює послідовності табуляцій та пробілів одним пробілом і виконує пошук слів у реченнях через створені об'єкти.

### 🎵 Лабораторна робота №5: Наслідування та поліморфізм
* **Варіант:** 17
* **Формули варіанта:**
  * $C_{13} = 17 \pmod{13} = 4$ (Ієрархія музичних композицій)
* **Опис завдання:** Створення абстрактного класу `Track` та його спадкоємців (`RockTrack`, `PopTrack`, `JazzTrack`). Реалізовано клас `Album` (музичний диск), який дозволяє підрахувати загальну тривалість треків, відсортувати композиції за їхнім музичним стилем, а також знайти трек, що відповідає заданому діапазону тривалості.

### 📦 Лабораторна робота №6: Власні колекції (Generics)
* **Варіант:** 17
* **Формули варіанта:**
  * $C_2 = 17 \pmod 2 = 1$ (Інтерфейс колекції: `Set`)
  * $C_3 = 17 \pmod 3 = 2$ (Внутрішня структура: Двозв'язний список)
* **Опис завдання:** Створення типізованої (`Generic`) колекції `DoublyLinkedSet`, яка імплементує інтерфейс стандартного `Set`, використовуючи логіку двозв'язного списку. Реалізовано всі методи інтерфейсу, користувацький ітератор з методом `remove()`, а також 3 типи конструкторів (порожній, з одним об'єктом, зі стандартною колекцією). Колекція протестована на об'єктах з Лабораторної №5.

### 🚀 Додаткова лабораторна робота №1: Stream API
* **Варіант:** 17 (На основі Лабораторної №3)
* **Опис завдання:** Модифікація логіки обробки даних за допомогою Java Stream API. Створено pipeline обробки колекції спортивного інвентарю з використанням щонайменше 3 конвеєрних методів (`filter()`, `sorted()`, `peek()`) та термінального методу (`findFirst()`).
